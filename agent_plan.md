# 基于现有 RAG 编排 Agent 的方案与实施计划

## 文档目的与范围
本文档用于指导在现有 RAG 系统之上构建上层 Agent 的工程实现，目标是在保持离线检索可用的前提下，引入 LangChain 与 LangGraph 的编排能力，完成面向法律场景的交互式检索与报告生成，并提供可扩展的在线搜索与多智能体协作能力。文档面向后续的编辑与编码工作，强调可落地、可扩展与可维护，并尽量对结构、职责与接口约定给出明确描述。

## 系统目标与设计原则
系统目标是让主 Agent 负责用户交互与意图路由，同时能够按需调用 RAG 检索、在线搜索与专业报告生成能力，最终输出包含证据来源的合规回答或结构化报告。设计上坚持“检索与编排解耦、证据与结论对齐、离线优先与在线补充”的原则，使得模型能力迭代或工具替换时不会破坏核心检索链路的稳定性，并确保任何法律结论都可追溯到明确的证据来源。

## 总体架构概述
整体架构由三层组成：主 Agent 交互层、工具与检索层、多智能体报告层。主 Agent 使用 LangGraph 的 `create_agent` 作为基础工具调用图，承担对话、路由与调度职责；工具与检索层以现有 `rag_search` 为主，并补充在线搜索工具；多智能体报告层参考 deerflow 结构，将研究、归纳、写作与质检解耦为多个角色，在同一任务上下文中协作产出专业报告。

## 主 Agent 职责与交互流程
主 Agent 接收用户问题后，首先进行意图识别并形成路由决策，随后根据路由选择调用 RAG 或在线搜索工具，并在必要时触发多智能体报告流程。主 Agent 在任何输出阶段都必须显式绑定证据来源，并在证据不足时给出不确定性说明与追问请求，以保证法律场景的严谨性与可解释性。

## 工具与接口约定
RAG 检索工具沿用现有 `rag_search` 接口，输入包含 `query`、`topk`、`db_name`、`is_hyde` 等字段，输出为包含证据与 `source` 的结构化文本。在线搜索工具提供统一接口，内部可根据模型自主决策选择 SERPAPI 或 TAVILY，其中鉴权与网络访问均由环境变量 `SERPAPI_API_KEY` 与 `TAVILY_API_KEY` 控制。多智能体报告工具对外表现为一个可调用的“专业法律顾问团队”入口，主 Agent 通过该入口获取 Markdown 报告正文与引用索引。

## 多智能体报告生成结构
报告生成参考 deerflow 的分工方式，核心角色包括协调与规划、资料检索与归纳、要点提炼与论证、报告写作与格式化、最终质检与合规校验。各角色之间通过共享任务上下文与证据池进行协作，协调角色负责将主 Agent 的路由与证据输入拆解为可执行子任务，并在完成后将统一的报告草案交回主 Agent 进行最终输出与交互封装。

## 模型与运行环境策略
模型调用采用 ModelScope 兼容 OpenAI API 的方式，主 Agent 以 `ChatOpenAI` 作为统一入口，并保留 `base_url` 与 `api_key` 的环境变量配置，以兼容不同的推理后端。RAG 相关能力要求离线可用，在线搜索与报告增强允许联网调用，确保在无网络或无 API Key 的情况下仍能完成基础检索问答。

## 引用规范与输出格式
所有结论性内容必须包含证据来源，引用格式应与 RAG 输出的 `source` 对齐，并在报告中清晰标注来源路径与法条定位信息。建议输出采用“论点—证据—结论”结构，以避免结论脱离证据，且在多来源时使用统一的引用编号或内联 `source=` 标记，确保读者可快速回溯到原始材料。

## 工作流编排与状态管理
LangGraph 的状态以消息列表与证据池为核心，主 Agent 运行过程中持续积累 `messages` 与 `observations`，并在关键节点生成 `route`、`evidence` 与 `final_answer` 等中间字段。工作流在工具调用后回到模型节点进行综合归纳，直到模型不再提出工具调用并生成最终答复，从而保证主 Agent 的行为与工具调用链路保持可追踪与可重复。

## MCP 与 Skills 的扩展思路
MCP 与 Skills 作为可选扩展能力，应在不破坏现有检索链路的前提下引入，优先用于外部知识库、任务管理与结构化输出的场景。接入时需明确权限边界与失败回退策略，避免工具不可用导致主 Agent 全链路失败，并在输出中显式标识外部工具的使用痕迹与证据来源。

## 实施路径与交付物
实施过程建议分为基础编排、在线增强与多智能体报告三个阶段推进。基础编排阶段完成主 Agent 与 `rag_search` 的稳定整合、意图路由与证据引用规范；在线增强阶段补充 SERPAPI 与 TAVILY 工具并完善联网回退策略；多智能体报告阶段完成 deerflow 风格的角色拆分、上下文共享与报告合成，最终形成可复用的报告生成能力。交付物包括主 Agent 入口脚本、工具层封装、配置与环境变量说明，以及可直接运行的示例流程。

## 风险与应对
主要风险包括模型输出与证据不一致、在线搜索不稳定、工具调用过度与成本不可控等。应对策略是在输出层强制引用规范、为在线搜索设置超时与重试机制、对工具调用进行最小必要化约束，并在无法确认结论时优先给出不确定性提示与追问。

## 完成标准
完成标准应体现为功能可用与输出可追溯：主 Agent 能稳定完成法律类问题的检索与回答，输出中包含明确证据来源；在线搜索可在需要时被模型正确调用且可用；多智能体报告能够生成结构化 Markdown 报告并满足引用规范，且在无网络环境下仍能保持离线检索可用的基本能力。
